# recipe-graph — connectivity tooling

Measures the pack's design north star: *"if you drew a line from every item to what it's used for, it
should be one or two cohesive webs, not many disconnected clusters."* Builds a graph from the recipe
digests (`../mod-data/recipes/*.txt`) **plus** the live `kubejs/` overlay and reports how connected the
pack is. Full math + interpretation: [`docs/CONNECTIVITY.md`](../../docs/CONNECTIVITY.md).

## Files
| File | What |
|---|---|
| `recipe_graph_lib.py` | shared core — graph build, components, depth-to-spine, cut-vertices, mod projection |
| `recipe-graph.py` | CLI — prints the markdown connectivity report |
| `recipe-graph-viz.py` | generates `recipe-web.html` (interactive offline mod map) |
| `recipe-web.html` | the generated viz (committed; CI regenerates it — see below) |

## Usage
```sh
python3 tools/recipe-graph/recipe-graph.py                 # report (no-vanilla lens + kubejs overlay)
python3 tools/recipe-graph/recipe-graph.py --with-vanilla  # keep minecraft: nodes (upper bound)
python3 tools/recipe-graph/recipe-graph.py --jars-only     # untouched base game
python3 tools/recipe-graph/recipe-graph.py --remove minecraft,create   # how Create's addons weave alone
python3 tools/recipe-graph/recipe-graph-viz.py             # rebuild recipe-web.html
```
Open `recipe-web.html` in any browser (no deps): drag nodes, toggle mods, or use the **Remove vanilla** /
**Remove Create spine** presets and watch the giant-web % recompute live.

## Notes
- Counts Create **parts and methods** — a modded recipe-type (`create:mixing`, `occultism:ritual`, …) is a
  node linking everything it produces (vanilla types dropped as universal).
- Pure-stdlib Python 3; no third-party deps. Re-run after any recipe change — it doubles as a regression check.
- **`recipe-web.html` is regenerated by CI** (`.github/workflows/recipe-web.yml`) on changes to the recipes,
  digests, or these scripts; don't hand-edit it.
