# connectivity — the pack's web, measured

*(Renamed from `tools/recipe-graph/` in #129, when mob drops joined recipes as graph input — old paths in issues/docs predating that point here.)*

Measures the pack's design north star: *"if you drew a line from every item to what it's used for, it
should be one or two cohesive webs, not many disconnected clusters."* Builds a graph from the recipe
digests (`../mod-data/recipes/*.txt`) **plus** the live `kubejs/` overlay and reports how connected the
pack is. Full math + interpretation: [`docs/CONNECTIVITY.md`](../../docs/CONNECTIVITY.md).

## Files
| File | What |
|---|---|
| `connectivity_lib.py` | shared core — graph build, components, depth-to-spine, cut-vertices, mod projection |
| `connectivity.py` | CLI — prints the markdown connectivity report |
| `connectivity-viz.py` | generates `connectivity-web.html` (interactive offline mod map) |
| `connectivity-web.html` | the generated viz (committed; CI regenerates it — see below) |

## Usage
```sh
python3 tools/connectivity/connectivity.py                 # report (no-vanilla lens + kubejs overlay)
python3 tools/connectivity/connectivity.py --with-vanilla  # keep minecraft: nodes (upper bound)
python3 tools/connectivity/connectivity.py --jars-only     # untouched base game
python3 tools/connectivity/connectivity.py --remove minecraft,create   # how Create's addons weave alone
python3 tools/connectivity/connectivity-viz.py             # rebuild connectivity-web.html
```
Open `connectivity-web.html` in any browser (no deps): drag nodes, toggle mods, or use the **Remove vanilla** /
**Remove Create spine** presets and watch the giant-web % recompute live.

## Notes
- Counts Create **parts and methods** — a modded recipe-type (`create:mixing`, `occultism:ritual`, …) is a
  node linking everything it produces (vanilla types dropped as universal).
- **Mob drops are edges too (#129, default-on; `--no-drops` to exclude).** Entity loot tables from
  `../mod-data/loot/` become mob nodes (`mod:entities/<mob>`) with a *direct* edge to each dropped item —
  no shared "drop" hub, which would weld every mob into one artificial cluster. The report's
  **drop-only items** list (drops no recipe touches) is the mob-curation signal (#90/#94). Caveats: the
  digest is referenced-ids only, so these are *possible* drops (presence), **not rates**; block self-drops
  and chest/structure loot (#18) are excluded by table type; and under `--with-vanilla` a few vanilla
  *entity* ids can appear as pseudo-drops (`minecraft:blaze`) — unfilterable without losing real drops
  whose item shares the entity id (`minecraft:chicken`). Modded refs are validated against the by-mod
  block/item registries, so that blur is vanilla-lens-only. The `organic/mob` bucket is viz coloring,
  not connectivity — real drop edges superseded it.
- Pure-stdlib Python 3; no third-party deps. Re-run after any recipe change — it doubles as a regression check.
- **`connectivity-web.html` is regenerated by CI** (`.github/workflows/connectivity-web.yml`) on changes to the recipes,
  digests, or these scripts; don't hand-edit it.
